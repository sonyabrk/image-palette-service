# Image Palette Service

Микросервис на Go для анализа цветовой палитры изображений.
Принимает картинку - возвращает палитру, доминирующий цвет
и координаты настроения в двумерном эмоциональном пространстве.

Разработан как самостоятельный сервис и как расширение для
платформ визуального вдохновения (например, для автоматического
определения mood-координат при загрузке изображений).

---

## Что делает сервис

Загружаешь картинку -> получаешь:

- **Палитру** из N доминирующих цветов в формате HEX
- **Доминирующий цвет** — самый распространённый в изображении
- **mood_x** — минимализм (0) -> максимализм (1)
- **mood_y** — тёмное (0) -> светлое (1)
- **brightness** — средняя яркость изображения (0–1)
- **saturation** — средняя насыщенность (0–1)

---

## Как работает алгоритм

### Извлечение палитры - Median Cut

Алгоритм рассматривает все пиксели изображения как точки
в трёхмерном пространстве RGB. Задача - найти N цветов,
которые лучше всего описывают картинку.

### Координаты настроения

**mood_y** (тёмное -> светлое) — средняя яркость палитры в
цветовой модели HSL. Тёмные тона дают значение близкое к 0,
светлые — к 1.

**mood_x** (минимализм -> максимализм) — комбинация трёх факторов:

| Фактор | Вес | Описание |
|--------|-----|----------|
| Цветовое разнообразие | 40% | Сколько значимо различающихся цветов в палитре |
| Контраст | 35% | Разброс яркости между самым тёмным и светлым цветом |
| Насыщенность | 25% | Средняя насыщенность всех цветов палитры |

### Ограничения алгоритма

Сервис анализирует **цветовые характеристики**, а не содержимое.
Он не видит объекты, текстуры и композицию.

Тёмная фотография с богатой деталировкой получит низкий mood_y
независимо от сложности содержимого - это особенность
цветового подхода. Для полного анализа визуальной
сложности потребуется подключение CV-модели.

## Запуск

### Переменные окружения

Создай файл `.env` в корне проекта:

```env
PORT=8080
WORKERS=4
CACHE_TTL_SECONDS=300
MAX_CLUSTERS=8
```

| Переменная | По умолчанию | Описание |
|---|---|---|
| PORT | 8080 | Порт сервиса |
| WORKERS | 4 | Количество воркеров в пуле |
| CACHE_TTL_SECONDS | 300 | Время жизни кэша в секундах |
| MAX_CLUSTERS | 8 | Количество цветов в палитре |

### Без Docker

```bash
go run ./cmd/server
```

### Через Docker

```bash
# Собрать и запустить
docker-compose up --build

# Запустить в фоне
docker-compose up --build -d

# Остановить
docker-compose down
```

## API

### POST /analyze

Анализирует изображение и возвращает палитру с координатами настроения.

**Запрос:** `multipart/form-data`, поле `image`

Поддерживаемые форматы: JPEG, PNG, GIF, WEBP. Максимальный размер: 10MB.

**Пример запроса:**

```bash
curl -X POST http://localhost:8080/analyze \
  -F "image=@/path/to/your/photo.jpg"
```

**Коды ответа:**

| Код | Причина |
|-----|---------|
| 200 | Успешно |
| 400 | Поле image отсутствует, неверный формат или файл > 10MB |
| 500 | Внутренняя ошибка сервера |

---

### GET /health

Проверка живости сервиса. Используется Docker и системами оркестрации.

```bash
curl http://localhost:8080/health
```

```json

```
